# Request Tracking

## Description

This feature allows residents to monitor the status of their submitted rescue request in real time using a unique tracking code issued upon submission. It provides transparency and communication between the resident and their assigned responder throughout the rescue process. Residents can also view the live location of the assigned rescue team on the map during deployment.

---

## User Roles Involved

- Resident (User)

---

## User Stories

### Resident

#### Story 1: Track Rescue Request

As a resident,
I want to track the status of my submitted rescue request,
so that I know whether help is on the way and what to expect next.

**Acceptance Criteria**

- [ ] User can enter their tracking code on a tracking page to view their request
- [ ] Tracking page displays the current status of the request (Pending, In Progress, Completed)
- [ ] User receives real-time status updates without needing to refresh the page
- [ ] User is notified (on-screen alert) when their request status changes
- [ ] Tracking page displays the detected needs from the triage algorithm with their corresponding icons
- [ ] Tracking page displays an estimated response time (if available)
- [ ] System displays an error message if the entered tracking code is invalid or not found
- [ ] No login is required. Access is granted through the unique tracking code only or clicking the floating status bar in the browser where the request was made

---

#### Story 2: View Live Rescue Team Location

As a resident,
I want to see the rescue team's location on the map in real time,
so that I know how far they are and when to expect them.

**Acceptance Criteria**

- [ ] Tracking page displays a map showing the resident's location and the rescue team lead's current location
- [ ] Rescue team lead's location updates in real time without needing to refresh the page
- [ ] A route path is drawn from the team lead's current location to the resident's location
- [ ] Map is only shown when a rescue team has been assigned and deployment is active
- [ ] Map displays an appropriate message when no team has been assigned yet
- [ ] No login is required to view the live location

---

#### Story 3: Communicate with Assigned Responder

As a resident,
I want to send and receive messages with my assigned responder,
so that I can provide additional information and receive updates during the rescue.

**Acceptance Criteria**

- [ ] User can view all messages in a shared chat thread including messages from the admin and the assigned team lead
- [ ] Each message is labeled with the sender role (Admin, Team Lead, Resident)
- [ ] User can send messages through the chat interface on the tracking page
- [ ] Chat updates in real time without needing to refresh the page
- [ ] Chat is only available once a responder has been assigned to the request
- [ ] System displays an appropriate message if no responder has been assigned yet
- [ ] No login is required. Chat access is granted through the unique tracking code only

---

## Notes (Optional)

- Tracking code is issued automatically upon successful submission of a rescue request.
- No login is required to access the tracking page, ensuring accessibility during emergencies.
- Detected needs displayed on the tracking page are generated by the triage algorithm and may be overridden by the admin.
- Live rescue team location is only broadcast during active deployment and stops automatically when the request is marked as completed.
